What are the benefits of version control? Version control is a very important aspect of web development, allowing multiple users to modify files and merge changes at the same time. It also allows you to easily archive successive versions of version-controlled items, storing the data that distinguishes one version from another. The easiest way to explain this to someone who knows nothing about git (for example) would be to imagine an essay on a single text document. If a dozen people wanted to contribute to the essay without a version control system, it would be a slow process, with all individuals having to work separately. Also, changes could be made to the master document without the knowledge or approval of the documents owner. All of this is alleviated with VCS - Version control stores the date and time on which the item was created, when it was checked out or checked in, and the user who performed the action. It also allows you to easily recover from accidental deletions, and save disk space.

How does git help you keep track of changes? Git is a version control system. When you make changes, Git helps you save them. Here are the steps: Firstly, in the Max OS Terminal, check where you are by typing “git branch”. This will let you know if you’re in the master. Next, type “git status”. This will let you know if anything is being tracked or modified, ready to commit. Next “git pull” which fetches from and integrates with another repository or a local branch. From here, we create a new branch using “git checkout -b ”. This allows us to work on a “branch” of the master in our working tree, without modifying the original code. We then create a new directory by typing “mkdir ”. Inside the new directory we create a new file by typing “touch ”. From there, we navigate into the Sublime text editor by typing “sublime ”. WORK WORK WORK! When finished, we save the Sublime file and go back to Terminal and type “git status”. It should show that we have a modified which we then need to “git add ” to be staged. If we put in “git status” now, we’ll see that the file has been staged and is ready to commit. The second to last step is typing “git commit -m” and adding a good commit message between quotation marks. Finally we do “git push origin and then head into GitHub!

Why use GitHub to store your code? GitHub, and VCSs in general are the mechanism you use to store code in a safe place. One of the best features of GitHub is “forking” which allows you to copy someone else’s code to modify yourself. If you make changes you’d like to share, you can send a notification called a “pull request” to the original owner. That user can then, with a click of a button, merge the changes found in your repository with the original repo. These three features – fork, pull request and merge – along with the Facebook-like usability, are what make GitHub so powerful. Since Git encourages granular recording of changes (commit early - commit often!), programmers, from absolute beginners (me) or experts, can trace the steps of some of the greatest developers in the world and find out how they solved tricky problems. This (and more) is why everyone should store their code on GitHub!
