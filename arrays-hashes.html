Arrays and Hashes - Technical Blog

Posted on February 15th 2016 by andrew crowley

Ruby looks at collections of objects by putting them inside container objects. In Ruby, there are two main built-in classes we will be dealing with: arrays and hashes.

Arrays are ordered, integer-indexed collections of any object. Ruby array indexing starts at 0. A negative index is assumed to be relative to the end of the array; that is, an index of -1 indicates the last element of the array, -2 is the next to last element in the array, and so on. Arrays can contain different types of object - integers, strings, floats, etc. An array’s job is simply to store other objects.

Hashes are also ordered collections. Hashes store objects in pairs, each pair consisting of a key and a value. You retrieve a value by using the key. They are similar to arrays, but where an array uses integers as its index, a Hash allows you to use any object type. Keys are unique per hash: you can only have one key/value pair for any given key.

Arrays and hashes are closely connected. An array is, in a sense, a has where the keys happen to be consecutive integers.

Here are some examples of arrays:

array = [“platinum”, “gold”, “silver”]

second_array = [1, "two", 3.0]

i_wanna_visit = [“Maldives”, “Africa”, “New Zealand”, “Thailand”]

In the case of the bottom array, “puts i_wanna_visit[1]"" would print “Africa” to the screen.

And here are some examples of hashes:

fave_cologne = Hash.new {"Paul" => "Acqua di Gio", "Johnny" => "Polo Sport", "Randy" => "Brut"}

hash = {“best“ => “platinum”, “better” => “gold”, “good” => “silver”}

grades = { "Johnny" => 90, "Billy" => 88, "Jenny" => 94 }

To output the entire last hash, you would type "puts grades" or you could output a key by calling its value. graders["Billy"] would output "88."
